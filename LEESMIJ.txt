FIXLI — WEBAPP/MVP
==================

ZO ZET U HET LOKAAL AAN HET WERK
1. Pak het hele zip-bestand uit (belangrijk: de map "js" moet naast de
   HTML-bestanden staan — open index.html dus NIET rechtstreeks vanuit
   het zip-venster, maar pak eerst alles uit).
2. Open index.html in uw browser. Klaar.

ZO ZET U HET OP GITHUB PAGES
1. Upload alle bestanden (incl. de map js/) naar de root van uw repository.
2. Zet GitHub Pages aan. Alle links zijn relatief, dus het werkt ook
   onder een submap zoals /fixli/.
3. Bezoekers met oude data van een eerdere versie: de app migreert hun
   localStorage automatisch bij het laden.

DEVELOPMENT-MODUS
Voeg ?demo=1 toe aan een URL (bijv. vakman-dashboard.html?demo=1) om
demo-snelkoppelingen en hints zichtbaar te maken. Normale bezoekers
zien deze niet.

PLACEHOLDER
Het admin-e-mailadres staat overal als placeholder: info@voorbeeld.nl
Vervang dit vóór livegang:
- js/store.js → DEFAULT_SETTINGS.adminEmail
- contact.html, privacybeleid.html, klachtenprocedure.html en de footer
  (zoek-en-vervang "info@voorbeeld.nl")

NOG TE KOPPELEN VOOR LIVEGANG (zoek op ">> BACKEND" en "// BACKEND" in de code)
- Echte database + authenticatie (localStorage is een tijdelijke demo-opslag,
  wachtwoorden staan nu onversleuteld op het apparaat van de gebruiker)
- Stripe/Mollie/iDEAL voor betaallinks en uitbetalingen
- E-mailservice (Resend, SendGrid, Mailgun) voor de notificaties
- Geocoding voor echte afstandsberekening
- Juridische teksten laten controleren (staan als concept gemarkeerd)

TESTSCENARIO'S (allemaal werkend)
A. Vaste prijs:  aanvragen → admin keurt goed → vakman accepteert →
   klantgegevens zichtbaar → extra kosten (klant-akkoord + Fixli-controle) →
   afronden → factuur met betaallink → klant betaalt → uitbetaling → review
B. Uurtarief:    aanvragen (uurtarief + min. 1 uur zichtbaar) → accepteren →
   uren invullen → automatisch herberekend → factuur o.b.v. uren
C. Op aanvraag:  aanvragen → admin maakt prijsvoorstel → klant accepteert →
   klus beschikbaar voor vakmannen
